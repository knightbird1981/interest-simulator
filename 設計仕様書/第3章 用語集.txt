# 第3章 用語集

## 3.1 目的

本章では、本設計仕様書全体で使用する用語を統一的に定義する。

第1章 1.9.1で定めたとおり、同一の意味で複数の表現を使用してはならない。本章で定義した名称を全章で使用する。

---

## 3.2 業務用語

| 用語 | 定義 |
|---|---|
| 積立元本／元本 | 初期投資額と毎月積立額の累計。運用利益（利息）を含まない。 |
| 評価額 | 元本に運用益を加算した運用資産総額。 |
| 満期年月 | 生年月日と満期年齢から算出する、利用者が満期年齢に達する年月（YYYY-MM）。日は考慮しない（第10章 10.5参照）。 |
| 運用期間／投資月数（investmentMonths） | 積立開始年月から満期年月までの月数。Ver.1.2より、利用者の直接入力ではなく生年月日・満期年齢・積立開始年月から自動算出する（第10章 10.5参照）。 |
| 満期年齢 | 利用者が資産運用を終了させたい年齢。運用期間入力モードが①の場合に利用者が入力する項目（第7章参照）。 |
| 運用期間入力モード | 運用期間の算出方法を「①生年月日から算出」「②運用期間から算出」のいずれかに切り替える設定。モードにより表示・必須となる入力項目が異なる（第6章 6.4.7、第7章 7.3.4参照）。 |
| 運用利益／利益 | 評価額－元本で算出される金額。負の値も許容する。 |
| 手取り利益 | 利益から税額を差し引いた税引後利益。 |
| 想定年利／年利 | 利用者が入力する年率の利回り（%）。 |
| 実効月利／月利 | 年利から算出される月単位の利率。複利選択時は年利を複利換算した値（単純な年利÷12ではない）、単利選択時は年利÷12の単純な値（第10章 10.6参照）。 |
| 複利 | 利息にも翌月以降利息が付与される運用方式。運用方法（interestType）の選択肢の一つ（第10章 10.4.1参照）。 |
| 単利 | 元本のみに利息が発生し、利息自体には利息が付かない運用方式。運用方法（interestType）の選択肢の一つ。Ver.1.6で追加（第10章 10.4.1、10.7.2、10.9.2参照）。 |
| 積立頻度 | 積立を行う周期。Ver.1.0では「毎月」のみ対応する。 |
| 運用方法 | 資産運用の計算方式。「複利」「単利」のいずれかを利用者が選択する（Ver.1.6より単利を追加）。税方式のような両方式同時算出は行わず、選択された1方式のみを計算する（第10章 10.4.1参照）。 |
| 源泉分離課税 | 運用益に対し一律20.315%（所得税15.000%＋復興特別所得税0.315%＋住民税5.000%）を課す税方式。 |
| 所得税方式 | 運用益を一時所得として単体で単純計算する税方式。総収入金額（評価額）から支出金額（元本）と特別控除額（最高50万円）を差し引いた額の1/2を課税所得とし、所得税速算表に基づき所得税を算出、これに復興特別所得税（0.315%）と住民税（10%）を加算する（第10章 10.16参照）。 |
| 一時所得 | 対価性のない一時的な所得の区分。本システムでは運用益をこの区分として単体で単純計算し、他の所得と合算する総合課税は考慮しない。 |
| 特別控除額 | 一時所得の計算で差し引く控除額。MIN(500,000円, 利益)で算出する。 |
| 復興特別所得税 | 課税所得に対し0.315%の固定税率で課す税。源泉分離課税・所得税方式の両方に適用する（第10章 10.14参照）。Ver.1.0では固定値とし、税率入力機能は設けない。 |
| 税方式トグル | 画面上で「源泉分離課税」「所得税方式」の表示を切り替えるUIコントロール。計算結果の再計算は行わない（第6章 6.4.6参照）。 |
| 税金比較結果 | 源泉分離課税・所得税方式の双方について算出した税額・手取り利益の一覧。 |
| 年次データ | 12か月ごとに生成される元本・評価額・利益・税額等の集計データ。 |
| 月次データ | 1か月ごとに生成される元本・評価額・利益・税額等の集計データ。グラフ表示に使用する。 |
| 円未満切り捨て | 金額計算の結果に対しfloor()を適用する丸め方式。四捨五入は使用しない。 |

## 3.3 画面・機能用語

| 用語 | 定義 |
|---|---|
| シミュレーション画面 | 積立条件の入力・計算実行・結果表示を行う本システム唯一の画面（SCR-001）。 |
| シミュレーション条件 | 計算実行前に利用者が入力する条件一式（初期資産・毎月積立額・想定年利・生年月日・満期年齢・積立開始年月・積立頻度・運用方法）。運用期間（投資月数）はこれらから自動算出するため、直接入力する条件には含まない。 |
| シミュレーション結果 | 計算実行後に表示される最終資産・元本・運用益・利益率等の結果一式。 |

---

## 3.4 データ・インターフェース用語

| 用語 | 定義 |
|---|---|
| 引数オブジェクト（SimulationInput） | runSimulation()関数に渡す入力データの形式。第12章参照。 |
| 戻り値オブジェクト（SimulationResult） | runSimulation()関数が返す結果データの形式。summary・monthlyData・yearlyData・taxComparisonを含む。第12章参照。 |
| taxComparison | 税方式ごと（源泉分離課税／所得税方式）の税額・手取り利益を格納した配列。 |

---

## 3.5 略語・記号

| 略語 | 正式名称 | 用途 |
|---|---|---|
| SCR | Screen | 画面IDの接頭辞（例：SCR-001） |
| VAL | Validation | バリデーションエラーコードの接頭辞（例：VAL-001） |
| F- | Function | 機能IDの接頭辞（例：F-001） |

---

## 3.6 未確定用語（要確認）

Ver.1.6時点で未確定の用語はない。

---

## 3.7 表記統一ルール

以下のとおり表記を統一する。誤って別表現を用いた場合は本章の表記に修正する。

| 統一表記 | 使用禁止表現の例 |
|---|---|
| 評価額 | 現在資産、保有額 |
| 運用利益／利益 | 収益、儲け |
| バリデーション設計 | 入力チェック設計 |
| 計算ロジック設計 | 計算仕様 |
| 処理インターフェース設計 | API設計 |

---

## 3.8 他章との関連

用語集は全章と関連する。各章で新たな用語を定義した場合は、本章への追記を必須とする。
